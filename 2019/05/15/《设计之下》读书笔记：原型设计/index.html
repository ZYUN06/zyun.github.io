《设计之下》读书笔记：原型设计

发表于 2019-05-15 | 分类于 读书笔记 |

原型

即 Prototyping，一般分为三个阶段：概念设计、低保真原型和高保证原型。

概念设计

  1. 概念是点子的载体。
  2. 培养视觉化思维，当有一个好的点子蹦出来，直接画出相对应的示意图。

低保真原型的形式

原型图（流程图和框架图）、纸原型、实体原型。

插播其他感兴趣的小点儿

  1. 为什么要显示性别：性别是除了头像以外，另一个十分吸引人的元素，提升了内容上的节奏感。并且通过简单的符号便可以表达，何乐而不为呢？
  2. “写评论按钮”在阅读圈（搜狐新闻客户端中的“阅读圈”）中十分重要，是用户除阅读以外，最主要的行为，也是大多数用户在浏览过程中唯一的操作行为。
    写评论的需求，一般发生在阅读之后，所以我们将它设计在原文的下端，界面的右侧，便于点击。并且由于评论源于“写评论按钮”，所以使用它来分割内容和评论在逻辑上也较为合理。
  3. 时间元素统一设计在每一条分享的右上端，较为独立，也不那么引人注目。
  4. 评论居于最下端的好处：评论数是一个不可控数据，对于“没有评论”或者“评论特别多”的极端情况，“居于最下”不会破坏视觉焦点，张弛度较大。
  5. 对于字数很多的评论，可用弹窗、区域滚动或进入一个新的界面等方式来显示完整内容。但这些网页上的传统交互方式都会在某种程度上中断阅读体验，在较小的手机屏幕上也不便操作。
  6. 平板电子设备的特性：较大的屏幕尺寸、丰富的手势操作、灵活的界面组成等。
  7. 实体原型可避免老年人面对新兴电子设备时容易产生的抵触心理。

高保真原型

  1. 即交互文档，这是将产品需求视觉化的第一个过程。
  2. 交互文档中不必要的视觉细节会让观看的人不自觉地从视觉的层面上进行思考。这样会给观看的人增加额外的认知负担，更糟糕的是，可能在评审时引出“节外生枝”的讨论，如：为啥这个头部是蓝色的？我觉得红色比较适合。
  3. 文档中，各元素的比例应尽量与实际比例相近。越接近真实比例的线框图越可以帮助交互设计师对之后对视觉效果有个评估，避免到了视觉设计阶段遇到问题再返工。
  4. 文档中带有指向性的箭头和线段：箭头有圆头的那一端放在哪就表示点击此处，箭头尖指向的是点击后跳转的页面。只有一头是圆点的线头表示对某处进行说明。

插播其他感兴趣的小点儿

  1. 设计时把希望用户先注意到的内容放在比较明显的位置，把相关联的内容放在相邻的顺序上，才能有效地展现内容，引导用户按照你的期望去使用产品。
  2. 市场分析、用户调研和反馈、内容运营的需要、高层的指示都是常见的需求产生源头。
  3. 在各种限制中寻求平衡和突破是我们常常在做的事。
  4. 在设计时，充分考虑可扩展性。
